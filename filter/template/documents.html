Documents

Purpose

This project aims to enhance user convenience by analyzing whether certain text requires censorship and censoring it using the ChatGPT API. It functions as a text censorship service that goes beyond conventional structured filtering to censor text that cannot be filtered by existing means and even offensive text. Despite the need for censorship of inappropriate content, it often goes unfiltered.

How to use?

- Python

0. It may run on Python.

1. Install Filchatter. (Click code to copy)

pip install filchatter

2. Import Filchatter. (Click code to copy)

from Filchatter import Filchatter

3. Get API key at website. Get API key

4. Create Filchatter instance and set your API key. (Click code to copy)

client = Filchatter()
client.set_api_key("api_key")

5. Filter chat with send_json_request() mehtod. (Click code to copy)

client = Filchatter()
client.set_api_key(api_key)
chat = "Hello World!"
filter_result_json = client.send_json_request(chat)
print(filter_result_json.json()['result'])

send_json_request() method requires String type value as first parameter. And then return json type result looks as follows:

{
    "result": "some value of result"
}

If you use invalid api key, may get error looks as follows:

{
    "error": "Invalid api_key"
}

Methods

This part describes all the methods we provide. Instance of Filchatter() can use this methods.

1. set_api_key

set_api_key(String api_key)

Set API key of instance to first parameter

2. get_api_key

get_api_key(String name)

It returns the API key created with the first parameter. If you send empty String name, you may get error looks as follow:

{
    "error": "Name not provided"
}

3. send_json_request

send_json_request(String chat, Integer moderation_option)

It receive first parameter and modify it with Chat GPT. And return json data looks as follow:

{
    "result": "some value of result"
}

If your instance of Filchatter is set with an incorrect API key, you may get error looks as follow:

{
    "error": "Invalid api_key"
}

You can send optional parameter as second parameter. It's default value is 0, that is use API for prompt mode. If you set this 1, you can use moderation mode and return json data looks as follow:

{
    'result': True,
    'result_dict': {
        'harassment': False,
        'harassment_threatening': False,
        'hate': False,
        'hate_threatening': False,
        'self_harm': False,
        'self_harm_instructions': False,
        'self_harm_intent': False,
        'sexual': False,
        'sexual_minors': False,
        'violence': False,
        'violence_graphic': False,
        'self-harm': False,
        'sexual/minors': False,
        'hate/threatening': False,
        'violence/graphic': False,
        'self-harm/intent': False,
        'self-harm/instructions': False,
        'harassment/threatening': False
    }
}
